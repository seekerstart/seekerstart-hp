# ポーカー鳳凰戦：リーグ制度・設計メモ（Season 4 構想）

> 作成日: 2026-06-05 / ステータス: 設計合意済み（実装はS4が近づいてから）
> 経緯: トップページのS3リーグ表示更新をきっかけに、S4以降のリーグ制度・鳳凰位条件を壁打ちで再設計した結論。

## 🎯 北極星
- **最強の6-max を決める**。最高位の称号「**鳳凰位**」を設け、いずれ**歴代6名**が揃えば「伝説の6max」とする。

## 1. サーバー / リーグ構造（Season 4 から）
- **S4 で2サーバーに分割**: **AB サーバー（上位）** と **CD サーバー（下位）**
- **AB サーバー = プール100人**: **A=30人 / B=70人**
- **CD サーバー = 残り全員**: **C = 101位以下の上位40% / D = それ以下**
- 人数は **A < B < C < D**（例: 300人時 → A30 / B70 / C80 / D120）
- ⚠ このピラミッドは **総勢約300人が前提**（約163人では C<B で崩れる）

## 2. 順位の決め方
- 指標 = **総獲得BB（BB単位）＋ 最低ハンド数ゲート**（未達は対象外）
- **サーバー内（A/B、C/D）** = 毎季の順位で **再スライス**（上位30=A 等）→ 自動
- **サーバー間（AB↔CD）** = S5以降に通常の昇降格（具体人数 X は未確定）
- スケール時の根拠: サーバーを分けると総BBの横比較が不能になるため、**サーバー内相対順位**で判定する

## 3. 鳳凰位
- **鳳凰の羽**: **A 在籍者が、その季「鳳凰卓（= 上位サーバー top-6）」に入ると1枚**
- **羽 通算3枚で鳳凰位**（N=3）
- **A 在籍ゲートあり**: B 在籍者は top-6 に入っても羽なし（昇格はする）。→「まず A に上がる」下積みを要求
- **永久称号・歴代制**（初代鳳凰位…）、**認定後も競争継続**（天鳳・将棋の永世称号に倣う）
- **S3 の羽**: 現 A リーグの3名（なんでやねん / ElNino / takatony）が S3 で全体 top-6 に入ったら獲得。S3 で出る羽は最大3枚（種火）。本格スタートは S4 から

## 4. S3 → S4 の初期配置（立ち上げ・今回限り）
- **S3 の結果のみ**で全員を順位付け（累計ではない）
- 上位100 → AB（A=1〜30位 / B=31〜100位）、101位以下 → CD（C=上位40% / D=残り）
- **床保護**: 既存ユーザーは S3 結果で**現リーグより下がらない**（S3成績は"上げ方向"にだけ作用）
- 結果として **C/D 飛び級は自動**（S3で全体上位なら C・D からでも A・B へ）。**飛び級はこの立ち上げ時のみ**、S5以降は通常運用
- 最低ハンド数ゲートで「少数ハンドの上振れ」を抑制
- リスク評価: 床保護により「実力者が1季の下振れで落ちる」事故は防げる。残る運リスク（上振れ昇格）は S5の降格＋羽3枚（累積）で自己修正 → **許容**

## 5. Season 3（目前に開始）の扱い
- **S3 は現行のまま走る**（単一サーバー、現状の緩い A3/B5/C155/D ラベル）。S4 で結果ベースに再編されるので S3ラベルは直さなくてよい
- トップページは**所属選手のみ更新済み**（A=3 / B=5 / C=上位20名 / D=「今シーズンから参加したプレイヤー全員」）＋ 昇降格ルールカードの高さ揃え
- **昇降格ルールの文面・config/seasons.json は未変更**（S4 が近づいてから改訂）

## 6. 将来のスケール（〜1000人）
- 2層思想: **サーバー = 流動性 / リーグ = 格付け**
- 鳳凰戦は週1・月曜20-23時の固定イベントなので同時接続率が高く、サーバーは比較的小さくても成立
- 人口増に応じて **サーバーを増設**（例: AB / C / D / E / F の5サーバー、％ベースのピラミッド）
- 「何サーバー/何リーグ開くか」は**人口が決める**

## 7. やらないこと
- **隠れ bot は不採用**（競技・称号・スポンサーの信用リスク）。流動性は構成側（割らない/束ねる/卓を絞る）で吸収
